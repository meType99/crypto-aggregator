# Как дать ссылку, чтобы сайт открывался у любого человека

Есть **2 способа**. Выберите подходящий:

| Способ | Ссылка работает | Нужен включённый ПК | Сложность |
|--------|-----------------|---------------------|-----------|
| **ngrok** | Пока запущен у вас | Да | 5 минут |
| **Render.com** | 24/7 в интернете | Нет | 15 минут |

---

## Способ 1 — ngrok (быстро показать одногруппникам)

Подходит, если нужно **прямо сейчас** скинуть ссылку на пару часов.

### Шаг 1. Запустите сайт

```powershell
cd "c:\Users\roman\OneDrive\Рабочий стол\практика\project"
venv\Scripts\activate
python run.py
```

Должно появиться: `Running on http://127.0.0.1:5000`

### Шаг 2. Установите ngrok

1. Скачайте: https://ngrok.com/download
2. Распакуйте `ngrok.exe`
3. Зарегистрируйтесь на ngrok.com (бесплатно)
4. В личном кабинете скопируйте authtoken и выполните:

```powershell
ngrok config add-authtoken ВАШ_ТОКЕН
```

### Шаг 3. Откройте туннель (второе окно PowerShell)

```powershell
ngrok http 5000
```

Появится строка вида:

```
Forwarding   https://abc123.ngrok-free.app -> http://localhost:5000
```

### Шаг 4. Скиньте ссылку

Отправляйте людям: **https://abc123.ngrok-free.app**

> Пока у вас запущены `python run.py` и `ngrok` — сайт доступен всем. Выключили ПК — ссылка перестанет работать.

## Способ 2 — Render.com (постоянная ссылка 24/7)

Подходит для **практики, отчёта, портфолио** — сайт работает без вашего компьютера.

### Шаг 1. Залейте проект на GitHub

1. Создайте репозиторий на https://github.com/new (например `finrate`)
2. В PowerShell:

```powershell
cd "c:\Users\roman\OneDrive\Рабочий стол\практика\project"
git add .
git commit -m "FinRate: агрегатор курсов валют и криптовалют"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/finrate.git
git push -u origin main
```

### Шаг 2. Создайте сервис на Render

1. Зайдите на https://render.com и зарегистрируйтесь
2. **New +** → **Web Service**
3. Подключите GitHub-репозиторий `finrate`
4. Настройки (Render подхватит из `render.yaml` автоматически):

| Поле | Значение |
|------|----------|
| Build Command | `pip install -r requirements.txt` |
| Start Command | `gunicorn backend.app:app --bind 0.0.0.0:$PORT --workers 1 --timeout 120` |
| Plan | Free |

5. Нажмите **Create Web Service**
6. Подождите 3–5 минут — статус станет **Live**

### Шаг 3. Скиньте ссылку

Render даст URL вида:

```
https://finrate-xxxx.onrender.com
```

Эту ссылку может открыть **любой человек** с телефона или компьютера.

> На бесплатном тарифе сервер «засыпает» после 15 мин без посетителей. Первое открытие может занять 30–60 сек — это нормально.

## Что проверить после публикации

Откройте в браузере:

- `https://ваша-ссылка/` — главная страница
- `https://ваша-ссылка/api/currencies` — JSON с валютами
- `https://ваша-ссылка/api/crypto` — JSON с криптовалютами

## Частые проблемы

| Проблема | Решение |
|----------|---------|
| ngrok: «connection refused» | Сначала запустите `python run.py`, потом ngrok |
| Render: build failed | Проверьте, что `requirements.txt` в корне репозитория |
| Render: долго грузится | Free-тариф — первый запрос после сна медленный |
| Нет данных криптовалют | CoinGecko иногда блокирует запросы — подождите 5 мин |
| Ссылка localhost не работает у других | `localhost` виден только на вашем ПК — нужен ngrok или Render |

- **Постоянная ссылка для отчёта** → Render + GitHub
